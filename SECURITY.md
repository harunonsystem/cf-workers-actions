# Security Policy

## Reporting Security Vulnerabilities

If you discover a vulnerability in this repository, please do not open a public issue.
Instead, report it via direct message to @harunonsystem on X.

We take security seriously and will respond as quickly as possible.

## Supported Versions

Security updates are provided for the latest version only.

## Security Best Practices

When using these GitHub Actions:

- Use the latest version of actions
- Review the code before using in production
- Use appropriate permissions for your workflows
- Regularly audit your dependencies
